OG Image Generator Documentation

Overview

The OG Image Generator allows you to create custom Open Graph images for your website. You can specify various parameters to customize the image according to your needs.

URL

The url for all images will be the same:

https://osig.app/g/

Parameters

Parameters vary by style. All of them are optional, but a title and subtitle usually produce a better preview.

Below is a list of all parameters we currently support:

  • key: Your API key. Can be found in your settings page, if you are registered.
  • style: Image style (default: "base")
    • - base
    • - logo
  • site: This dictates the size of the image (default: "x")
    • - x (1600 by 900)
    • - meta (1200 by 630)
  • font: Font to use for text (more will be added)
    • - helvetica
    • - markerfelt
    • - papyrus
  • title: Main title text
  • subtitle: Subtitle text
  • eyebrow: Eyebrow text
  • image_url: URL of the background image

Usage

The url for all images will be the same:

<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:image" content="{ generated_image_url }"/>

Using the MCP

OSIG also exposes a Model Context Protocol server for agent clients that need to inspect image options, render previews, and build signed image URLs.

  1. Get your profile key from the settings page.
  2. Connect your MCP client to the hosted Streamable HTTP endpoint: https://osig.app/mcp/
  3. Authenticate requests with either X-API-Key: <profile-key> or Authorization: Bearer <profile-key>.
  4. Start with get_image_generation_contract to see the available styles, fields, dimensions, and recommended workflow.

If your MCP client accepts a JSON-style server definition, the hosted connection looks like this:

{
  "mcpServers": {
    "osig": {
      "url": "https://osig.app/mcp/",
      "headers": {
        "X-API-Key": "YOUR_OSIG_PROFILE_KEY"
      }
    }
  }
}

A good agent workflow is:

  • Call normalize_image_params to canonicalize style, font, size, and text fields.
  • Call render_image_preview while iterating on title, subtitle, eyebrow, and image URL.
  • Call build_signed_image_url when the preview is ready to publish.
  • Use the signed URL in your Open Graph and Twitter image tags.

For local development with the repository checked out, run the stdio server instead:

uv run python mcp_server.py

Examples

Base Style

This is useful for general pages like Articles, Tutorial, News pages. Pro tip, you can generate the background image using AI. Those work super well.

Here is what gets generated for my site's articles:

This is the URL that generates the image:

https://osig.app/g?
  site=x
  &style=base
  &font=helvetica
  &title=10 Years of Great Books
  &subtitle=I'm starting a 10 Year Reading Plan proposed in the 1st Volume (The Great Conversation) of the "Great Books of the Western World" series.
  &eyebrow=Article
  &image_url=https://www.rasulkireev.com/_astro/a_smiling_boy_from_Pixars_Coco.DrP3G0Ol.png

Logo Style

Logo style is useful when you want to highlight a company, a project, a job posting, or another branded page.

Here is how I use it for one of my job boards:

This is the URL that generates the image:

https://osig.app/g?
  site=x
  &style=logo
  &font=markerfelt
  &title=Narrative
  &subtitle=Founding Senior Software Engineer
  &image_url=http://res.cloudinary.com/built-with-django/image/upload/v1728024021/user-profile-image-prod/hhxfplsthiaytuttoamc.jpg

Fonts

  • Helvetica
  • Markerfelt
  • Papyrus
