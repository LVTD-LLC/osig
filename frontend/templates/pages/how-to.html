Docs

Publish OG images.

Use the generator for one image, or call the endpoint directly from your site. Most previews need a title, subtitle, template, and optional image URL.

Endpoint

All generated images are served from the same endpoint.

https://osig.app/g

Parameters

Parameters vary by template. Empty values still render, but useful text and an image source usually produce a better preview.

Name Use Common values
style Template to render. base, logo, job_classic, job_logo, job_clean
site Output dimensions. x: 1600 x 900, meta: 1200 x 630
font Font used for image text. helvetica, markerfelt, papyrus
title Main headline. Article title, job title, product name
subtitle Supporting text, truncated when needed. Summary, role detail, short description
eyebrow Small context label inside the generated image. Article, Remote, Launch, Guide
image_url Background image or logo URL. Public image URL
key Authenticated profile key from settings. Your OSIG token

Examples

Base style

Use the base style for articles, tutorials, essays, and general pages that have a strong cover image.

https://osig.app/g?
  site=x
  &style=base
  &font=helvetica
  &title=10 Years of Great Books
  &subtitle=I'm starting a 10 Year Reading Plan.
  &eyebrow=Article
  &image_url=https://example.com/cover.png

Logo style

Use the logo style for companies, projects, job posts, and pages where a brand mark matters more than a full background image.

https://osig.app/g?
  site=x
  &style=logo
  &font=markerfelt
  &title=Narrative
  &subtitle=Founding Senior Software Engineer
  &image_url=https://example.com/logo.png

Output controls

Use output controls when your publishing platform needs JPEG output or a smaller file.

  • format: png or jpeg. The default is png.
  • quality: 1 to 100. JPEG defaults to 85 when omitted.
  • max_kb: best-effort file size target, currently tuned for JPEG.
  • v: cache-busting version token. Change this when preview content changes.

Signed URLs

Signed URLs protect generated previews from tampering. Send the desired image parameters to the signing endpoint and embed the returned URL.

POST /api/sign
{
  "params": {
    "style": "logo",
    "site": "x",
    "title": "Narrative",
    "format": "jpeg",
    "quality": 80
  },
  "expires_in_seconds": 3600
}

Using the MCP

OSIG exposes a Model Context Protocol server for agent clients that need to inspect options, render previews, and build signed image URLs.

  1. Get your profile key from the settings page.
  2. Connect your MCP client to https://osig.app/mcp/.
  3. Authenticate with X-API-Key: <profile-key> or Authorization: Bearer <profile-key>.
  4. Call get_image_generation_contract first to inspect styles, fields, dimensions, and the recommended workflow.
{
  "mcpServers": {
    "osig": {
      "url": "https://osig.app/mcp/",
      "headers": {
        "X-API-Key": "YOUR_OSIG_PROFILE_KEY"
      }
    }
  }
}

A useful agent workflow is to normalize parameters, render previews while iterating, build a signed URL, then place that URL in your Open Graph and Twitter image tags.

normalize_image_params
render_image_preview
build_signed_image_url

For local development with this repository checked out, run the stdio server.

uv run python mcp_server.py

Meta tags

Use the generated image URL in Open Graph and Twitter card metadata.

<meta property="og:image" content="{ generated_image_url }" />
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:image" content="{ generated_image_url }" />

Validation checklist

  • Bump v when image content changes.
  • Regenerate a signed URL if signed parameters changed.
  • Deploy metadata with the new image URL.
  • Open social validators before sharing the page.

Start with the onboarding wizard when you want OSIG to generate meta tags and validation links for you.
